localStorageはどのような場面で使用しましたか？

フォーム入力やCRUDデータを、

一時的なクライアント側の保存用途です。

🇰🇷 한국어

localStorage는 어떤 상황에서 사용했나요?

👉 답변

페이지를 새로고침해도 CRUD 데이터나
작성 중이던 상태를 유지하기 위해 사용했습니다.
클라이언트 측 임시 저장 용도입니다.

② sessionStorage와의 차이는 무엇인가요?
🇯🇵 일본어

localStorageとsessionStorageの違いは何ですか？

👉 回答

localStorageはブラウザを閉じてもデータが保持され、
sessionStorageはタブを閉じると削除されます。
永続性が必要な場合はlocalStorageを選びます。

🇰🇷 한국어

localStorage와 sessionStorage의 차이는 뭔가요?

👉 답변

localStorage는 브라우저를 닫아도 데이터가 유지되고,
sessionStorage는 탭을 닫으면 사라집니다.
지속성이 필요한 경우 localStorage를 사용합니다.

③ 어떤 데이터를 저장했고, 어떤 데이터는 저장하지 않았나요?
🇯🇵 일본어

どのデータを保存し、どのデータは保存しませんでしたか？

👉 回答

CRUDの元データのみを保存し、
UI状態や一時的な検索条件は保存しませんでした。
責務を分けるためです。

🇰🇷 한국어

어떤 데이터는 저장하고, 어떤 데이터는 저장하지 않았나요?

👉 답변

CRUD의 원본 데이터만 저장했고,
검색어나 UI 상태 같은 임시 값은 저장하지 않았습니다.
역할을 분리하기 위해서입니다.

④ localStorage 사용 시 주의점은 무엇이라고 생각하나요?
🇯🇵 일본어

localStorage使用時の注意点は何ですか？

👉 回答

セキュリティ上、個人情報や認証情報は保存しません。
また、容量制限があるため大量データには不向きです。

🇰🇷 한국어

localStorage 사용 시 주의점은 뭐라고 생각하나요?

👉 답변

보안상 개인정보나 인증 정보는 저장하면 안 됩니다.
또한 용량 제한이 있어 대용량 데이터에는 적합하지 않습니다.

⑤ localStorage 대신 서버를 쓰지 않은 이유는?
🇯🇵 일본어

なぜサーバーではなくlocalStorageを使いましたか？

👉 回答

この機能では認証や共有が不要だったため、
クライアント側だけで完結する方がシンプルだからです。

🇰🇷 한국어

왜 서버가 아니라 localStorage를 사용했나요?

👉 답변

인증이나 데이터 공유가 필요 없는 기능이었기 때문에,
클라이언트에서만 처리하는 것이 더 단순했기 때문입니다.

⑥ localStorage를使わない方がいいケースは？
🇯🇵 일본어

localStorageを使うべきでないケースは？

👉 回答

認証情報や機密データ、
複数端末で同期が必要なデータの場合は使いません。

🇰🇷 한국어

localStorage를 쓰면 안 되는 경우는 언제인가요?

👉 답변

인증 정보나 민감한 데이터,
여러 기기 간 동기화가 필요한 경우에는 사용하지 않습니다.
